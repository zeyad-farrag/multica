## Approach (this repo)

1. Add cursor + limit helpers in `server/internal/handler/cursor.go`: `encodeCursor(t time.Time, id uuid.UUID) string`, `parseCursor(s string) (time.Time, uuid.UUID, bool, error)` (format `<RFC3339Nano>:<UUID>`, empty string ‚Üí "no cursor"), `parseLimit(r *http.Request, def, max int) (int, error)` (strict ‚Äî malformed or `>max` returns error so handlers can emit 400 `{"error":"invalid_limit"}`). Malformed cursor ‚Üí 400 `{"error":"invalid_cursor"}` (AC-3).
2. Add `ListIssuesUpdatedSince` to `server/pkg/db/queries/issue.sql`: `SELECT ... FROM issue WHERE workspace_id = $1 AND updated_at >= $2 AND ($3::timestamptz IS NULL OR (updated_at, id) > ($3, $4)) ORDER BY updated_at ASC, id ASC LIMIT $5`. Add `CountIssuesUpdatedSince` for the `total` field. Mirror the row shape of existing `ListIssues` so `issueListRowToResponse` can be reused (AC-1).
3. Run `make sqlc` from repo root; verify `server/pkg/db/generated/issue.sql.go` updates with no manual edits (AC-10).
4. In `server/internal/handler/issue.go` add `ListIssuesUpdatedSinceForWorkspace(w, r)`: read `chi.URLParam(r, "id")` for workspace UUID, parse `updated_since` (RFC3339, required, 400 if missing/malformed), parse optional `cursor` via helper, parse `limit` via helper (default 200, max 1000). Fetch `limit+1` rows; return first `limit` and set `next_cursor` only when `limit+1` were returned. Call `h.enrichIssuesWithLabels(ctx, resp)` on the response (matches existing pattern at the end of `ListIssues`). Reuse `issueListRowToResponse`. Return `{issues, next_cursor, total}`. Do NOT modify the existing `ListIssues` handler ‚Äî AC-2 requires byte-for-byte identical behavior on `/api/issues` when `updated_since` is absent.
5. Add `ListCommentsByAuthorTypeDate` to `server/pkg/db/queries/comment.sql`: `SELECT ... FROM comment WHERE workspace_id = $1 AND author_id = $2 AND type = $3 AND created_at >= $4 AND created_at < $5 AND ($6::timestamptz IS NULL OR (created_at, id) > ($6, $7)) ORDER BY created_at ASC, id ASC LIMIT $8`. Add `CountCommentsByAuthorTypeDate`. Do NOT add `author_type = 'member'` in SQL (AC-5).
6. In `server/internal/handler/comment.go` add `ListCommentsForBackfill(w, r)` with the AC-5 NOTE comment at the top. Validate `author_id` (UUID, required), `type` (non-empty, required), `date` (`YYYY-MM-DD`, required) ‚Äî 400 on missing/malformed. Compute the `[start, end)` UTC window (default UTC; if workspace `settings.work_week.timezone` or equivalent is set, use `time.LoadLocation`; if `LoadLocation` fails, fall back to UTC and continue). Reuse cursor helpers + `commentToResponse`. Skip reactions/attachments enrichment ‚Äî backfill only needs `(author_id, type, created_at, content)`. Return `{comments, next_cursor, total}`. Do NOT modify the existing per-issue `ListComments`.
7. Add `ListActivityByWorkspace` to `server/pkg/db/queries/activity.sql`: `SELECT ... FROM activity_log WHERE workspace_id = $1 AND (sqlc.narg('action')::text IS NULL OR action = sqlc.narg('action')) AND (sqlc.narg('actor_id')::uuid IS NULL OR actor_id = sqlc.narg('actor_id')) AND created_at >= $2 AND ($3::timestamptz IS NULL OR (created_at, id) > ($3, $4)) ORDER BY created_at ASC, id ASC LIMIT $5`. Add `CountActivityByWorkspace`.
8. In `server/internal/handler/activity.go` add `ListWorkspaceActivity(w, r)`: read workspace from URL param, parse `since` (RFC3339, required), optional `action` (free-form snake_case string, no allowlist) and `actor_id` (UUID), parse cursor + limit via helpers.
