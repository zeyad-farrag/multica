## Approach (this repo)

1. **Add cursor helpers** in `server/internal/handler/cursor.go`:
   - `parseCursor(s string) (time.Time, pgtype.UUID, error)` — splits on the LAST `:` (UUIDs contain hyphens, not colons; keep the LAST split to be robust to RFC3339Nano with offsets like `2026-04-27T11:21:38.123456789+00:00`). Empty string → zero values, no error (first page).
   - `encodeCursor(t time.Time, id pgtype.UUID) string` — `t.UTC().Format(time.RFC3339Nano) + ":" + uuidToString(id)`.
   - `parseLimit(raw string, defaultN, maxN int) (int, error)` — strict validation. Empty → default. Non-int / `<1` / `>maxN` → error.
   - `writeInvalidCursor(w http.ResponseWriter)` — emits `400 {"error":"invalid_cursor"}` per AC-3.

2. **Add sqlc queries** (do not modify existing queries):
   - `issue.sql` — `ListIssuesUpdatedSince` selects the same column list as `ListIssues` (so the existing `db.ListIssuesRow` shape can be reused via `issueListRowToResponse`); WHERE `workspace_id = $1 AND updated_at >= $2 AND ($3::timestamptz IS NULL OR (updated_at, id) > ($3, $4))` ORDER BY `updated_at ASC, id ASC` LIMIT `$5`. Use `sqlc.narg` style consistent with existing queries. Pair `CountIssuesUpdatedSince` for `total`.
   - `comment.sql` — `ListCommentsByAuthorTypeDate` selecting `*` (matches existing `ListCommentsSincePaginated` shape so `commentToResponse` can be reused); WHERE clauses `workspace_id`, `author_id`, `type`, `[start, end)`, `(created_at, id) > cursor`. Pair `CountCommentsByAuthorTypeDate`.
   - `activity.sql` — `ListActivityByWorkspace` selecting `*`; WHERE `workspace_id = $1 AND created_at >= $2 AND ($3::text IS NULL OR action = $3) AND ($4::uuid IS NULL OR actor_id = $4) AND ($5::timestamptz IS NULL OR (created_at, id) > ($5, $6))` ORDER BY `created_at ASC, id ASC` LIMIT `$7`. Pair `CountActivityByWorkspace`.

3. **Run `make sqlc`** to regenerate `server/pkg/db/generated/{issue,comment,activity}.sql.go`. Verify `go build ./...` succeeds. Do not hand-edit generated files (AC-10).

4. **Implement `ListIssuesUpdatedSinceForWorkspace`** in `server/internal/handler/issue.go`:
   - Read workspace ID via `chi.URLParam(r, "id")` then `parseUUID` (AC-1).
   - Strict-validate `updated_since` RFC3339; `cursor` via `parseCursor`; `limit` via `parseLimit(raw, 200, 1000)`. On any malformed → 400 (AC-3, AC-1 cap).
   - Query `ListIssuesUpdatedSince`, request `limit+1` rows; if returned `>= limit+1`, set `next_cursor = encodeCursor(rows[limit-1].UpdatedAt, rows[limit-1].ID)` and return only the first `limit`. Otherwise `next_cursor = nil`.
   - Map rows via existing `issueListRowToResponse` and call `h.enrichIssuesWithLabels(ctx, resp)` + `h.enrichIssuesWithLinks(ctx, resp)` to match `ListIssues` line 713-714 (AC-1 enrichment parity).
   - Fetch `CountIssuesUpdatedSince` for `total`.
   - Response shape: `{issues: [], next_cursor: *string, total: int64}`.
   - Add a top-of-handler comment: `// SPEC: §6.1 #3, §22 M-PR#3 — Story 1.4 read portion. Cursor format <RFC3339Nano>:<UUID>; fetch limit+1 to set next_cursor only when more exist.`
   - Leave the legacy `ListIssues` (line 598) untouched (AC-2 byte-for-byte).

5. **Implement `ListCommentsForBackfill`** in `server/internal/handler/comment.go`:
   - Read workspace ID from URL; require `author_id` (UUID), `type` (non-empty), `date` (`YYYY-MM-DD`); reject malformed → 400 (AC-4).
   - Compute `[start, end)` window: parse `date` in workspace TZ if `workspace.settings.timezone` is set, else UTC. Helper to fetch workspace TZ — use `h.Queries.GetWorkspace` and inspect `Settings` JSONB. Default UTC keeps the contract deterministic (Dev Notes "Workspace timezone").

## Test strategy (this repo)

- **Unit (cursor helpers)** — `server/internal/handler/cursor_test.go`:
  - encode/decode round-trip preserves nanosecond precision.
  - `parseCursor("")` returns zero values, no error.
  - `parseCursor("not-a-cursor")`, `parseCursor("garbage:not-a-uuid")`, `parseCursor("2026-04-27T00:00:00Z:not-a-uuid")` → error.
  - `parseLimit("", 200, 1000)` → 200; `parseLimit("0", ...)` / `parseLimit("-1", ...)` / `parseLimit("1001", ...)` / `parseLimit("abc", ...)` → error; `parseLimit("500", ...)` → 500.

- **Unit (issues)** — `server/internal/handler/issue_updated_since_test.go`:
  - Seed N=210 issues with two pairs sharing `updated_at` (tie-break case): page 1 returns 200, `next_cursor` non-nil; page 2 returns the remaining 10, `next_cursor` nil. AC-1, AC-3.
  - Cursor strict-after: with two issues at the same timestamp, `cursor=<ts>:<idA>` returns only `idB` (not idA). AC-3.
  - `limit=1001` → 400; `updated_since=not-rfc3339` → 400. AC-1, AC-3.
  - Missing PAT → 401 (use the existing test pattern that hits the full router). AC-7.
  - Non-member PAT → 403. AC-8.
  - **Regression**: `GET /api/issues` (no `updated_since`) returns the SAME body shape and rows as before this PR (snapshot the body or assert structure). AC-2.

- **Unit (comments)** — `server/internal/handler/comment_backfill_test.go`:
  - Seed comments by member A and agent B at three different timestamps in the same workspace day; filter by `(author_id=A, type=status_change, date=2026-04-27)` returns A's status_change rows only. AC-4.
  - Seed an agent-authored `status_change` row; assert it IS returned by `(author_id=<agent-id>, type=status_change, date=...)`. AC-5.
  - Pagination: seed 210 matching rows; verify cursor advances, no duplicates across pages. AC-4.
  - Missing `author_id`/`type`/`date` → 400. AC-4.

- **Unit (activity)** — `server/internal/handler/activity_workspace_test.go`:
  - Seed activity rows with `action='gate_bypassed'` and `details->>'outcome'='blocked'`; filter by `since + action=gate_bypassed` returns only matching rows. AC-6.
  - Cursor strict-after on tied timestamps. AC-6.
  - `actor_id` narrowing returns only that actor's rows. AC-6.
  - Missing PAT → 401. AC-7.

- **Integration** — `server/cmd/server/integration_test.go` (or new test file): boot the router with the test pool, mint a PAT for a workspace member, hit each new endpoint, assert 200 + body shape; without PAT → 401; as non-member → 403. AC-7, AC-8 (Subtask 4.4).

- **Manual verification**: none required — endpoints are programmatic. The standalone reconciler is the consumer and is built in a separate repo / story.

- **Regression — workspace settings.work_week pass-through**: extend `handler_test.go` (or workspace-specific test) to set `settings.work_week` on the test workspace and assert `GET /api/workspaces/{id}` returns it intact. Scope-boundary row 4.
